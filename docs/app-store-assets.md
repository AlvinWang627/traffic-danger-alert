# App Store 素材需求

## 📸 必需圖片素材

### 🎯 App 圖示 (App Icon)
- **尺寸**: 512x512 像素
- **格式**: PNG (32位元，無透明度)
- **設計要求**:
  - 清晰識別的安全/警示主題
  - 建議顏色：紅色警示 + 藍色科技感
  - 包含盾牌、警告標誌或道路元素
  - 避免過多文字，保持簡潔

### 🖼️ 功能圖片 (Feature Graphic)
- **尺寸**: 1024x500 像素
- **格式**: PNG 或 JPG
- **內容建議**:
  - App 名稱和主要功能展示
  - 手機使用場景 (駕駛時使用)
  - 「保護您的行車安全」等宣傳標語
  - 台北市地圖背景 (選用)

### 📱 手機截圖 (至少 2 張，最多 8 張)

#### 截圖 1: 主畫面 - 監控中狀態
- **內容**: 顯示「偵測中」狀態
- **要素**:
  - GPS 座標顯示
  - 「附近有 X 個危險路段」警示
  - 背景運作狀態
  - 效能監控資訊

#### 截圖 2: 設定畫面
- **內容**: 完整的設定選項
- **要素**:
  - 警示距離滑桿 (設定為 150m)
  - 聲音提醒開關
  - 背景運作設定
  - 電池優化選項

#### 截圖 3: 警示通知 (建議)
- **內容**: 通知出現的畫面
- **要素**:
  - 推播通知顯示
  - 「⚠️ 交通危險警報」標題
  - 距離和危險路段數量資訊

#### 截圖 4: 效能監控 (建議)
- **內容**: 效能詳細資訊畫面
- **要素**:
  - 記憶體使用量
  - 位置更新統計
  - 電池優化資訊

## 🎨 設計建議

### 顏色配置
- **主色調**: #2196F3 (藍色) - 科技感
- **警示色**: #F44336 (紅色) - 危險提醒
- **成功色**: #4CAF50 (綠色) - 安全狀態
- **背景色**: #FFFFFF (白色) - 清潔明亮

### 字體和文字
- 使用繁體中文
- 重要訊息使用粗體
- 保持足夠的對比度
- 避免過小的字體

### UI 元素
- 使用 Material Design 風格
- 圓角按鈕和卡片設計
- 清晰的圖示和符號
- 適度的陰影效果

## 📐 截圖規格要求

### 手機截圖
- **解析度**: 1080x1920 或 1440x2560 像素
- **格式**: PNG 或 JPG
- **方向**: 直向 (Portrait)
- **數量**: 2-8 張

### 平板截圖 (選用)
- **7 吋平板**: 1200x1920 像素
- **10 吋平板**: 1440x2560 像素
- **方向**: 直向或橫向皆可

## 🛠️ 製作工具建議

### 設計軟體
- **專業**: Adobe Photoshop, Illustrator
- **免費**: GIMP, Canva, Figma
- **線上**: Canva, Adobe Express

### 截圖工具
- **Android 模擬器**: Android Studio AVD
- **實體裝置**: 直接使用手機截圖功能
- **美化工具**: Device Art Generator, MockuPhone

## 🎯 截圖場景設定

### 場景 1: 日常使用
- 時間: 下午 2:30
- 位置: 台北市中心座標
- 狀態: 正在監控，顯示 2 個附近危險路段
- 電池: 80%

### 場景 2: 設定調整
- 顯示設定畫面
- 警示距離設為 200m
- 各種開關的不同狀態
- 說明文字清晰可見

### 場景 3: 效能展示
- 記憶體使用: 45MB
- 位置更新: 每 15 秒
- 電池優化: 啟用
- 運行時間: 2 小時 30 分

## 📋 製作檢查清單

### 圖示檢查
- [ ] 512x512 像素，PNG 格式
- [ ] 無透明背景
- [ ] 清晰可辨識
- [ ] 符合安全/導航主題

### 功能圖片檢查
- [ ] 1024x500 像素
- [ ] 包含 App 名稱
- [ ] 視覺吸引力強
- [ ] 傳達核心功能

### 截圖檢查
- [ ] 至少 2 張截圖
- [ ] 解析度符合要求
- [ ] 展示主要功能
- [ ] 文字清晰可讀
- [ ] 狀態列顯示合理時間和電量

### 整體檢查
- [ ] 所有文字使用繁體中文
- [ ] 顏色配置一致
- [ ] 設計風格統一
- [ ] 符合 Google Play 政策

## 💡 專業建議

### 截圖最佳實踐
1. **真實場景**: 使用實際的座標和時間
2. **清晰文字**: 確保所有文字都清晰可讀
3. **功能展示**: 每張截圖都要展示不同功能
4. **用戶視角**: 從使用者角度考慮最有吸引力的畫面

### 常見錯誤避免
- 避免截圖模糊或像素化
- 不要包含個人敏感資訊
- 避免過度美化導致與實際 App 差異過大
- 不要使用虛假或誇大的功能展示
