# Google Play Store 上架檢查清單

## 📋 上架前準備事項

### 🔧 技術要求

#### App Bundle 建置
- [ ] 建置 Android App Bundle (.aab 檔案)
  ```bash
  # Expo 建置命令
  eas build --platform android --profile production
  ```
- [ ] 設定正確的版本號 (versionCode 和 versionName)
- [ ] 確認目標 SDK 版本符合 Google Play 要求
- [ ] 測試不同螢幕尺寸和解析度
- [ ] 確認 ARM64 相容性

#### 安全性檢查
- [ ] 移除所有 console.log 和 debug 代碼
- [ ] 確認沒有硬編碼的 API 金鑰或敏感資訊
- [ ] 啟用程式碼混淆 (ProGuard/R8)
- [ ] 設定正確的網路安全配置

### 📱 App 內容準備

#### 圖示和截圖
- [ ] **App 圖示** (512x512 PNG, 高解析度)
- [ ] **功能圖片** (1024x500 PNG, 橫幅圖片)
- [ ] **手機截圖** (至少 2 張，最多 8 張)
  - 建議尺寸：1080x1920 或 1440x2560
  - 展示主要功能：
    - 主畫面（偵測中狀態）
    - 設定畫面
    - 警示通知範例
    - 效能監控畫面
- [ ] **平板截圖** (選用，7 吋和 10 吋)

#### 商店資訊
- [ ] **App 名稱** (最多 50 個字元)
  - 建議：「交通危險警示」或「台北交通安全助手」
- [ ] **簡短描述** (最多 80 個字元)
- [ ] **完整描述** (最多 4000 個字元)
- [ ] **關鍵字優化**：交通安全、行車警示、危險路段、GPS

### 🏷️ 分類和標籤

#### 應用程式分類
- [ ] **主要分類**：地圖與導航
- [ ] **次要分類**：汽車與車輛
- [ ] **內容分級**：適合所有年齡層
- [ ] **目標族群**：18-65 歲駕駛族群

#### 標籤設定
- [ ] 免費應用程式
- [ ] 包含廣告：否
- [ ] 應用程式內購買：否

### 📄 政策文件

#### 必要文件
- [x] **隱私權政策** (`docs/privacy-policy.md`)
  - [ ] 上傳至可公開存取的網址
  - [ ] 確認政策內容符合實際功能
- [ ] **服務條款** (建議新增)
- [ ] **資料安全問卷**
  - 位置資料：本地處理，不上傳
  - 裝置資訊：僅效能優化使用
  - 使用統計：匿名收集

#### 權限說明
準備詳細的權限使用說明：
- [ ] **位置權限**：核心功能必需，用於危險路段偵測
- [ ] **背景位置**：持續監控安全，可選功能
- [ ] **通知權限**：發送警示通知
- [ ] **震動權限**：震動提醒功能

### 🧪 測試要求

#### 功能測試

## ⚠️ 常見問題和注意事項

### 權限政策
- **位置權限**：需要明確說明使用目的和必要性
- **背景位置**：Google 對此權限審查嚴格，需要詳細說明
- **目標 SDK**：確保符合 Google Play 的最新要求

### 審查重點
- 隱私權政策必須詳細且可公開存取
- 功能描述要與實際功能一致
- 不可包含誤導性內容或虛假宣稱

### 發布時機
- 建議避開週末或假期發布
- 預留 1-3 天審查時間
- 準備緊急修復版本的發布流程
