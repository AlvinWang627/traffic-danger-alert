# App 發布指南

## 🚀 發布流程概覽

### 階段 1: 開發完成準備 (1-2 天)
1. 程式碼最終檢查和優化
2. 移除所有 debug 代碼
3. 設定正式版本號
4. 完整功能測試

### 階段 2: 建置和測試 (1 天)
1. 建置 production 版本的 AAB 檔案
2. 內部測試安裝和功能驗證
3. 效能和電池使用測試
4. 不同裝置相容性測試

### 階段 3: 素材準備 (2-3 天)
1. 設計 App 圖示和功能圖片
2. 製作應用程式截圖
3. 撰寫商店描述文案
4. 準備隱私權政策網頁

### 階段 4: Google Play Console 設定 (1 天)
1. 建立應用程式項目
2. 上傳 AAB 檔案
3. 填寫所有必要資訊
4. 設定發布軌道

### 階段 5: 審查和發布 (1-3 天)
1. 提交審查
2. 等待 Google 審查結果
3. 處理可能的審查意見
4. 正式發布

## 📋 詳細步驟指南

### Step 1: 程式碼準備

#### 版本設定
```javascript
// app.json
{
  "expo": {
    "version": "1.0.0",
    "android": {
      "versionCode": 1,
      "compileSdkVersion": 34,
      "targetSdkVersion": 34,
      "minSdkVersion": 26
    }
  }
}
```

#### 建置配置
```javascript
// eas.json
{
  "build": {
    "production": {
      "android": {
        "buildType": "app-bundle",
        "gradleCommand": ":app:bundleRelease"
      }
    }
  }
}
```

#### 程式碼清理檢查清單
- [ ] 移除所有 `console.log` 語句
- [ ] 移除 debug 相關代碼
- [ ] 確認沒有 TODO 或 FIXME 註解
- [ ] 檢查敏感資訊是否已移除
- [ ] 確認所有功能正常運作

### Step 2: 建置流程

#### EAS Build 設定
```bash
# 安裝 EAS CLI
npm install -g @expo/eas-cli

# 登入 Expo 帳號
eas login

# 初始化 EAS 配置
eas build:configure

# 建置 Android App Bundle
eas build --platform android --profile production
```

#### 本地測試
```bash
# 建置完成後下載 AAB 檔案
# 使用 bundletool 轉換為 APK 進行測試
java -jar bundletool.jar build-apks --bundle=app.aab --output=app.apks
```

### Step 3: Google Play Console 設定

#### 開發者帳號設定
1. **註冊 Google Play Console**
   - 前往 [Google Play Console](https://play.google.com/console)
   - 支付 $25 USD 註冊費
   - 完成身份驗證

2. **稅務和銀行資訊**
   - 設定稅務資訊 (台灣)
   - 設定銀行帳戶 (如有付費功能)

#### 應用程式建立
1. **建立新應用程式**
   ```
   應用程式名稱: 交通危險警示
   預設語言: 中文 (繁體)
   應用程式或遊戲: 應用程式
   免費或付費: 免費
   ```

2. **上傳 App Bundle**
   - 前往「版本管理」→「應用程式套件」
   - 建立新版本
   - 上傳 AAB 檔案
   - 填寫版本資訊

### Step 4: 商店資訊設定

#### 主要商店資訊
```
應用程式名稱: 交通危險警示
簡短說明: 保護您的行車安全！即時偵測附近危險路段，提供震動和聲音警示。
完整說明: [使用 google-play-store-description.md 的內容]
```

#### 圖片素材上傳
- 應用程式圖示: 512x512 PNG
- 功能圖片: 1024x500 PNG/JPG
- 手機截圖: 2-8 張，1080x1920 或 1440x2560

#### 分類設定
```
應用程式類別: 地圖與導航
內容分級: 適合所有年齡層
目標族群: 18-65 歲
```

### Step 5: 資料安全和隱私

#### 資料安全問卷
1. **位置資料**
   - 收集位置資料: 是
   - 資料類型: 精確位置、大概位置
   - 資料用途: 應用程式功能
   - 資料處理: 僅在裝置上處理

2. **裝置或其他 ID**
   - 收集: 否

3. **個人資訊**
   - 收集: 否

#### 隱私權政策
- 上傳 `privacy-policy.md` 到公開網址
- 在 Google Play Console 中連結該網址

### Step 6: 發布設定

#### 發布軌道建議
1. **內部測試** (建議)
   - 測試對象: 開發團隊
   - 測試期間: 2-3 天
   - 驗證所有功能正常

2. **封閉測試** (選用)
   - 測試對象: 親友或特定用戶
   - 測試期間: 1 週
   - 收集使用者回饋

3. **公開測試** (選用)
   - 公開招募測試者
   - 測試期間: 1-2 週
   - 大規模功能驗證

4. **正式發布**
   - 全面公開上架

#### 發布時程設定
```
階段式發布: 建議啟用
初始發布比例: 5%
每日增加比例: 20%
完全發布時間: 5 天
```

### Step 7: 審查準備

#### 審查重點項目
1. **隱私權合規**
   - 位置權限使用說明清楚
   - 隱私權政策完整且可存取
   - 資料處理方式透明

2. **功能完整性**
   - 所有宣稱功能都能正常運作
   - 無崩潰或嚴重錯誤
   - 使用者體驗良好

3. **內容適當性**
   - 無誤導性內容
   - 截圖與實際功能一致
   - 描述準確

#### 常見審查問題處理
1. **權限問題**
   - 準備詳細的權限使用說明
   - 確保每個權限都有明確用途

2. **隱私政策**
   - 確保政策可公開存取
   - 內容要與 App 功能一致

3. **功能說明**
   - 準備功能演示影片 (選用)
   - 提供詳細的使用說明

### Step 8: 發布後維護

#### 監控指標
- **安裝數據**: 下載量、安裝成功率
- **使用者評價**: 評分、評論內容
- **技術指標**: 崩潰率、ANR 率
- **效能數據**: 啟動時間、記憶體使用

#### 回應計畫
1. **用戶評論**
   - 24 小時內回應負面評論
   - 感謝正面回饋
   - 記錄改進建議

2. **技術問題**
   - 監控崩潰報告
   - 快速修復嚴重問題
   - 定期發布更新

#### 更新規劃
```
緊急修復: 24-48 小時
小更新: 2-4 週
大功能更新: 2-3 個月
```

## ⚠️ 注意事項和風險

### 常見拒絕原因
1. **權限濫用**: 要求不必要的權限
2. **隱私問題**: 隱私政策不完整或無法存取
3. **功能虛假**: 宣稱功能與實際不符
4. **技術問題**: 嚴重錯誤或崩潰

### 風險控制
- **多重測試**: 在不同裝置和 Android 版本測試
- **漸進發布**: 使用階段式發布降低風險
- **快速回應**: 準備緊急修復流程
- **備用方案**: 準備回滾計畫

### 時程預估
```
總時程: 7-14 天
- 準備階段: 5-7 天
- 審查等待: 1-3 天
- 發布執行: 1-2 天
- 監控調整: 持續進行
```
