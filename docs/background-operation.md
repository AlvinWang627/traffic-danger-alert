# 背景運作功能說明

## 概述

交通危險警報應用程式支援背景運作功能，確保即使在應用程式關閉或螢幕鎖定的情況下，仍能持續監控附近的危險路段並發送即時警報。

## 主要功能

### 1. 背景位置監控
- 使用 Expo Background Fetch 和 TaskManager
- 持續追蹤用戶位置變化
- 支援高精度和平衡精度模式
- 自適應位置更新頻率

### 2. 智能通知系統
- 背景發送推播通知
- 避免重複警報（30秒間隔）
- 根據移動狀態智能發送通知
- 支援聲音和震動提醒

### 3. 電池優化
- 自適應位置更新間隔
- 夜間自動減少更新頻率
- 移動偵測功能
- 可調整的距離閾值

## 技術實現

### 核心服務

#### BackgroundService
- 單例模式管理背景任務
- 處理位置權限和通知權限
- 註冊和管理背景任務
- 整合電池優化功能

#### BatteryOptimizationService
- 管理電池優化設定
- 計算最佳位置更新間隔
- 自適應精度調整
- 移動狀態偵測

### 背景任務

#### 位置監控任務 (background-location-task)
```typescript
TaskManager.defineTask(BACKGROUND_LOCATION_TASK, async ({ data, error }) => {
  // 處理位置更新
  // 檢查危險路段
  // 發送通知
});
```

#### 背景獲取任務 (background-fetch-task)
```typescript
TaskManager.defineTask(BACKGROUND_FETCH_TASK, async () => {
  // 定期檢查位置
  // 更新危險路段資料
  // 返回執行結果
});
```

## 設定選項

### 背景運作設定
- **啟用/停用背景運作**：控制是否在背景監控
- **狀態顯示**：顯示當前背景任務狀態

### 電池優化設定
- **啟用電池優化**：自動調整位置更新策略
- **位置更新間隔**：2-15分鐘可調整
- **距離閾值**：50-200公尺可調整
- **自適應精度**：根據移動狀態調整精度
- **移動偵測**：只在移動時發送通知

## 權限要求

### iOS
- `NSLocationWhenInUseUsageDescription`
- `NSLocationAlwaysAndWhenInUseUsageDescription`
- `NSUserNotificationsUsageDescription`
- `UIBackgroundModes`: `["location", "fetch", "background-processing"]`

### Android
- `ACCESS_COARSE_LOCATION`
- `ACCESS_FINE_LOCATION`
- `ACCESS_BACKGROUND_LOCATION`
- `RECEIVE_BOOT_COMPLETED`
- `WAKE_LOCK`
- `foregroundServiceType`: `["location"]`

## 電池優化策略

### 1. 時間基礎優化
- 夜間（22:00-06:00）自動減少更新頻率
- 根據用戶活動模式調整間隔

### 2. 精度自適應
- 靜止時使用平衡精度
- 移動時自動切換到高精度
- 根據移動速度調整更新頻率

### 3. 智能通知
- 只在移動時發送通知
- 避免重複警報
- 根據距離和時間過濾

### 4. 距離閾值
- 可調整的距離觸發閾值
- 較大閾值減少位置更新頻率
- 平衡準確性和電池使用

## 使用建議

### 最佳實踐
1. **啟用電池優化**：預設啟用以節省電池
2. **調整更新間隔**：根據使用場景調整
3. **使用自適應精度**：自動平衡準確性和電池使用
4. **啟用移動偵測**：減少不必要的通知

### 電池使用優化
- 在充電時可關閉電池優化以獲得最佳準確性
- 長時間靜止時建議增加更新間隔
- 夜間可考慮停用背景運作

## 故障排除

### 常見問題
1. **背景運作無法啟動**
   - 檢查位置權限是否已授予
   - 確認通知權限已啟用
   - 檢查設備的電池優化設定

2. **通知未收到**
   - 確認背景運作已啟用
   - 檢查通知權限設定
   - 確認設備不在勿擾模式

3. **電池消耗過快**
   - 啟用電池優化功能
   - 增加位置更新間隔
   - 使用平衡精度模式

### 除錯方法
- 查看控制台日誌
- 檢查背景任務狀態
- 監控位置更新頻率
- 驗證通知發送狀態
